# RTMP to HLS 스트리밍 트러블슈팅 가이드

본 문서는 라이브 스트리밍 서비스 구축 중 발생했던 주요 문제들과 그 해결 과정을 기록합니다.

## 1. 영상 송출 불가 및 검은 화면 (Black Screen) 이슈

### 문제 현상
- OBS에서 스트림을 보냈으나 브라우저에서 오디오만 들리고 영상은 검은 화면으로 나옴.
- FFmpeg 로그상에서는 정상적으로 처리되는 것처럼 보임.

### 원인 분석
- **ABR 분할 로직의 부작용**: `filter_complex`를 사용해 여러 화질로 쪼개는 과정에서 인코딩 설정이 복잡해져 초기 키프레임 생성이 늦어지거나 브라우저 호환성이 떨어지는 문제 발생.
- **픽셀 포맷 미지정**: 웹 브라우저 표준인 `yuv420p`가 아닌 다른 포맷으로 인코딩될 경우 일부 브라우저에서 재생 불가.

### 해결 방법
- **단일 화질(1080p)로 단순화**: MVP 단계에서는 ABR을 제거하고 소스 화질을 그대로 인코딩하도록 변경.
- **브라우저 호환성 옵션 강제**:
    - `-profile:v main`: 대부분의 디바이스 지원 프로필.
    - `-pix_fmt yuv420p`: 웹 표준 픽셀 포맷 지정.
    - `-g 60`: 2초(30fps 기준)마다 강제 키프레임 생성으로 세그먼트 정렬.

## 2. 스트리밍 시작 시 연결 초과 (Connection Timeout) 이슈

### 문제 현상
- 방송 시작 버튼을 누르면 첫 번째 시도에서 "연결 초과"로 실패하고, 두 번째 눌러야 정상적으로 스트림이 열림.

### 원인 분석
- **Race Condition**: OBS가 RTMP 서버로 데이터를 보내고 서버가 준비되기 전에 `FfmpegService`가 소스를 읽으려 시도함.
- **ffprobe 검증 부족**: 스트림이 열렸는지 확인하는 로직의 재시도 횟수(15회)와 타임아웃(3초)이 네트워크 지연을 감당하기에 부족했음.

### 해결 방법
- **스트림 확인 로직 강화 (`waitForStreamReady`)**:
    - **재시도 횟수 증가**: 15회 -> 20회.
    - **타임아웃 연장**: 3초 -> 5초.
    - **비디오 스트림 명시적 확인**: `ffprobe`에 `-select_streams v:0` 옵션을 추가하여 오디오만 있는 상태가 아닌 비디오 데이터가 실제로 들어올 때까지 대기하도록 수정.

## 3. 실시간 인제스트 불안정성

### 문제 현상
- 스트리밍 중 프레임 드랍이 발생하거나 속도(`speed`)가 1.0x 미만으로 떨어짐.

### 원인 분석
- **`-re` 옵션 오용**: `-re` (Read input at native frame rate) 옵션은 파일 기반 스트리밍에는 필수적이나, 이미 실시간인 RTMP 소스에는 사용하면 안 됨. 실시간 소스에 사용 시 입력 버퍼 딜레이를 유발함.

### 해결 방법
- **RTMP와 파일 스트리밍 로직 분리**:
    - `executeStreamingFfmpeg` (RTMP용): `-re` 옵션 제거, `-rw_timeout` 추가.
    - `executeFileFfmpeg` (파일용): `-re` 옵션 유지하여 실시간 속도로 송출.

## 4. VOD 자산화 실패 및 임시 파일 누적

### 문제 현상
- 스트리밍 종료 후 MP4 변환이 안 되거나, `/tmp/hls` 아래에 수많은 TS 파일이 남아 디스크 용량 차지.

### 해결 방법
- **`VodService` 구조 개선**:
    - **세션별 격리**: `sessionId/streamingName/file.mp4` 구조로 저장하여 파일 충돌 방지.
    - **자동 정리 (`cleanUp`)**: 변환 성공 여부와 관계없이 `finally` 블록에서 소스 TS 디렉토리를 재귀적으로 삭제하도록 구현.
