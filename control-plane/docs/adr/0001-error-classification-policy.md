# ADR-0001: 에러 분류 및 관리 정책 수립

## 상태
Accepted (승인됨)

## 맥락 (Context)
- 스트리밍 도메인은 실시간성 비즈니스 로직(세션 제어)과 인프라 제어(FFmpeg 프로세스)가 공존함.
- 에러 발생 시 원인이 비즈니스 논리 오류인지, 외부 프로세스 장애인지 즉각 식별하기 어려움.
- 클라이언트에게 단순한 HTTP 400/500 이상의 구체적인 실패 원인을 전달할 필요가 있음.

## 결정 (Decision)
에러 코드에 접두사(Prefix)를 사용하는 체계를 도입한다.
- **S (Session)**: 비즈니스 규칙 위반, 상태 전이 오류 등 (예: S-001)
- **P (Process)**: 외부 엔진(FFmpeg) 실행 실패, 좀비 프로세스 감지 등 (예: P-001)
- **메시지 포맷팅**: `StreamingErrorCode` 내부에 포맷팅 문자열을 포함하고, `StreamingDomainException` 생성 시 지연 포맷팅(String.format)을 적용하여 메모리 효율을 챙긴다.

## 결과 (Consequences)
- 에러 코드만 보고도 문제의 근원지(도메인 vs 인프라)를 즉시 파악 가능함.
- HTTP 상태 코드와의 매핑 정책(S는 400대, P는 500대)을 명확히 정의할 수 있음.

# 1. 에러 분류 체계

 분류 | 카테고리     | 설명                             | 주요 발생 위치                 
----|----------|--------------------------------|--------------------------
 S  | Session  | 스트리밍 세션의 생애주기 및 상태 전이 규칙 위반    | StreamingSession, Status 
 P  | Process  | FFmpeg 프로세스 제어, PID 관리, 실행 실패  | FfmpegExecutor           
 I  | Ingest   | RTMP 소스 끊김, 입력 신호 유실 (고도화 예정)  | LogMonitor               
 R  | Resource | CPU/메모리 초과, 디스크 용량 부족 (고도화 예정) | FileSystem               

## 1.1. 세션(S) 카테고리: "비즈니스 로직 및 상태 무결성"

세션의 흐름이 깨지는 이유를 명확히 분류하여 클라이언트의 잘못된 호출인지, 서버의 로직 오류인지 즉시 구분

 에러 코드 | 에러 명칭                      | 상세 설명 및 통계적 의미                                    
-------|----------------------------|---------------------------------------------------
 S-001 | SESSION_NOT_FOUND          | DB에 식별자가 없음. 잘못된 API 호출 혹은 데이터 삭제 추적용.            
 S-002 | DUPLICATE_STREAM_KEY       | 이미 송출 중인 키로 중복 시작 시도. 비정상적인 다중 송출 시도 통계.          
 S-003 | INVALID_STATUS_TRANSITION  | 상태 전이 규칙 위반. 시스템 로직 결함 혹은 비정상적 이벤트 순서 감지.         
 S-004 | SESSION_ALREADY_TERMINATED | 이미 종료(STOPPED/FAILED)된 세션에 대한 조작 시도. 레이스 컨디션 분석용. 

## 1.2. 프로세스(P) 카테고리: "엔진 실행 및 생명주기 관리"

FFmpeg 프로세스와 (혹은 외부) 관련된 ㅔㅇ러를 상세화하여 서버의 물리적 안정성과 엔진의 신뢰도 측정

 에러 코드 | 에러 명칭                 | 상세 설명 및 통계적 의미                                       
-------|-----------------------|------------------------------------------------------
 P-001 | PROCESS_START_FAILED  | FFmpeg 실행 명령어 오류 혹은 바이너리 부재. 배포/환경 설정 결함 지표.         
 P-002 | PROCESS_ORPHANED      | DB엔 생존으로 되어 있으나 실제 PID가 없음. '좀비 프로세스' 및 프로세스 증발율 계측. 
 P-003 | PROCESS_TIMEOUT       | 프로세스 기동 후 일정 시간 내 데이터 감지 실패. 엔진 초기화 지연 통계.           
 P-004 | PROCESS_ABNORMAL_EXIT | 의도치 않은 프로세스 종료(Crash). 엔진의 안정성 및 인코딩 부하 분석용.         

# 2. 로깅 및 예외 처리 정책

- 에러 로그 필수 포함 정보
    - `sessionId`: 문제 추적을 위한 고유 키
    - `errorCode`: 분류된 에러 코드
    - `context`: 에러 발생 시점의 데이터
- 예외 전파: 도메인 레이어에서 발생한 StreamingDomainException은 GlobalExceptionHandler에서 공통 처리
